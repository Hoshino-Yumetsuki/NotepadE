# Sign-Off Items & Risk Register

## Items Requiring USER SIGN-OFF Before Build

1. **Acrylic substitute** — static blurred-tint + opacity tokens; wallpaper-sampling host-backdrop dropped. (Phase 0 decision, Phase 7 build.)
2. **Write-restriction divergence — APPROVED (user, 2026).** UWP's block on system32 / `.bat`/`.cmd` associations were sandbox artifacts, not product intent; the rewrite is deliberately *more capable* (Node fs, no re-imposed block). Approved under the same reasoning as #10 (UWP platform artifacts are not parity requirements). DoD: (a) saving to a path UWP blocked succeeds; (b) opening/associating .bat/.cmd works; (c) session stores absolute paths re-validated on load; (d) one-line release note documents divergence. Genuine OS-permission failures still surface as the normal save-error path.
3. **Perf targets (SR-8)** — cold start ≤2000ms / RAM ≤250MB / installer ≤150MB / 1MB open ≤300ms are honest Electron ceilings, not UWP parity. Confirm acceptable.
4. **CM6 visual deltas — APPROVED (user, 2026).** CodeMirror 6 caret/selection/line-rendering will not be pixel-identical to the UWP RichEditBox; golden-image threshold ≤0.1% is the accepted tolerance. Where CM6 cannot match exactly (caret blink timing, native selection handles, IME candidate UI), the CM6-native behavior is accepted and documented. Editor-surface goldens are subject to R8 (UWP reference captures) before final visual sign-off.
5. **Regex flavor deltas — APPROVED (user, 2026).** Find/replace uses JS RegExp; .NET-only constructs are handled as: RightToLeft (find-previous) via the forward-match-all + pick-last-before-cursor shim (0.F); balancing groups, `\Z`/`\z`, .NET named-group syntax `(?<name>)` differences documented as known divergences. Acceptance = regex-parity fixture passes incl. the RTL reverse-search row; documented flavor gaps are accepted, not blockers.
6. **Smart Copy semantics — APPROVED (user, 2026).** Replicate the ACTUAL source behavior: Smart Copy (setting, default off) trims leading/trailing whitespace from the selection on COPY only (cut never trims). NOT "paragraph expansion" (that was a bundle error corrected against source). DoD: with Smart Copy on, copying a selection with surrounding whitespace places the trimmed text on the clipboard; cut is unaffected.
7. **F5 datetime format — APPROVED (user, 2026).** F5 inserts the date/time using the OS current-culture default format (source behavior via CultureInfo.CurrentCulture), NOT a fixed string. The fixed `"h:mm tt M/dd/yyyy"` format applies ONLY to the `.LOG` auto-timestamp (once per open). DoD: F5 output matches the OS locale's default datetime; `.LOG` first-line auto-stamp uses the fixed format.
8. **Other substitutes (batch sign-off):** compact-overlay → frameless alwaysOnTop; FutureAccessList → re-validated absolute paths (missing = unavailable tab); markdown-it; JS diff lib; middle-click autoscroll → OS-native; jump list → OS recents; share → OS share/clipboard; title bar → `titleBarStyle:hidden`+overlay.
9. **Out-of-scope confirmation:** Game Bar widget, Store/AppCenter telemetry (no-op shims), multi-line handwriting.
10. **File-size cap dropped — APPROVED (user, 2026).** UWP's 1,024,000-byte open limit was an architectural/performance artifact (RichEditBox + sandbox), not product intent. The rewrite drops the hard cap; opening files larger than the original is explicitly acceptable. Deliberate "more capable than 1:1" divergence (same category as #2). DoD: a file above the old boundary opens/edits/saves with 0% byte round-trip mismatch; no hard refusal. Soft safeguards for very large files (defer optional decorations) are allowed but must never refuse the file.

## Scope Ledger (settled)

**IN-SCOPE (full behavior parity):** tabbed editing/SetsView; all editor commands + shortcuts; find/replace (incl. regex + find-previous); encoding read/write/convert + EOL; session snapshot + crash recovery; cross-window full-state drag; status bar (8-column flyouts); all settings panes + live theme + accent; markdown preview + diff viewer (functional); print; 29-locale i18n; file associations + `notepads://` + CLI argv/cwd; multi-instance broker; custom title bar. (NOTE: the UWP 1,024,000-byte file-size cap is intentionally DROPPED — see sign-off item #10, not a parity requirement.)

**IN-SCOPE-WITH-SUBSTITUTE (documented divergence + sign-off):** host-backdrop acrylic → static blurred-tint; compact-overlay F12 → frameless alwaysOnTop; FutureAccessList → absolute paths re-validated; markdown engine → markdown-it; diff engine → JS diff lib; middle-click autoscroll → OS-native; jump list → OS recents; share → OS share/clipboard; title bar → titleBarStyle:hidden+overlay.

**OUT-OF-SCOPE:** Game Bar widget; Store/AppCenter telemetry (no-op shim); multi-line handwriting; RichEditBox internal quirks as such (replicate observable behavior only); Win32 read-only-file write exploit.
